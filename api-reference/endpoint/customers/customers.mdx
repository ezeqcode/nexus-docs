## Como utilizar este endpoint?

Esse endpoint converte um cliente de teste em um cliente ativo, definindo o período de ativação com base nos créditos utilizados.

### Headers obrigatórios

<ParamField
  header="Authorization"
  type="string"
  placeholder="Bearer {token}"
  required
>
  O token de autenticação no formato Bearer é obrigatório para acessar este endpoint.
</ParamField>

### Parâmetros da URL

<ParamField path="id-client" type="string" placeholder="505" required>
  ID do cliente a ser ativado. Esse parâmetro deve ser informado diretamente na URL.
</ParamField>

### Parâmetros do corpo da requisição (Payload)

<ParamField body="username" type="string" placeholder="123456" required>
  Nome de usuário do cliente a ser ativado.
</ParamField>

<ParamField body="password" type="string" placeholder="123456" required>
  Senha do cliente.
</ParamField>

<ParamField body="credits" type="number" placeholder="1" required>
  Quantidade de créditos para ativação, onde cada crédito corresponde a um período específico:
  - **0.5 Crédito** → 15 dias  
  - **1 Crédito** → 1 mês  
  - **2 Créditos** → 2 meses  
  - **3 Créditos** → 3 meses  
  - **5 Créditos** → 180 dias  
  - **10 Créditos** → 360 dias  
</ParamField>

<ParamField body="whatsapp" type="string" placeholder="(12) 3 1231-2312">
  Número de WhatsApp do cliente. Campo opcional.
</ParamField>

<ParamField body="country" type="string" placeholder="Brasil" required>
  País de origem do cliente.
</ParamField>

<ParamField body="notes" type="string" placeholder="Observações">
  Observações adicionais sobre o cliente. Campo opcional.
</ParamField>

### Exemplo de Requisição

```json
PATCH https://api.revenda.nexus/lines/active/505
Headers:
{
  "Authorization": "Bearer seu_token_aqui",
  "Content-Type": "application/json"
}
Payload:
{
  "username": "123456",
  "password": "123456",
  "credits": 1,
  "whatsapp": "(12) 3 1231-2312",
  "country": "Brasil",
  "notes": "Ativação oficial"
}
```

### Resposta

<ResponseField name="status" type="number">
  Indica o código HTTP da resposta. Esperado: 201 (Created).
</ResponseField>

<ResponseField name="response" type="object">

<Expandable title="Mostrar objeto">
<ResponseField name="id" type="number">
  Identificador único do cliente.
</ResponseField>

<ResponseField name="username" type="string">
  Nome de usuário do cliente.
</ResponseField>

<ResponseField name="password" type="string">
  Senha do cliente.
</ResponseField>

<ResponseField name="owner" type="number">
  ID do proprietário responsável pelo cliente.
</ResponseField>

<ResponseField name="expDate" type="string">
  Data de expiração do período ativo do cliente.
</ResponseField>

<ResponseField name="status" type="number">
  Status da conta (1 para ativo).
</ResponseField>

<ResponseField name="isTrial" type="number">
  Indica se o cliente é de teste (0 para cliente ativo).
</ResponseField>

<ResponseField name="whatsapp" type="string">
  Número de WhatsApp associado ao cliente.
</ResponseField>

<ResponseField name="country" type="string">
  País de origem do cliente.
</ResponseField>

<ResponseField name="createdAt" type="string">
  Data de criação do cliente.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Data da última atualização do cliente.
</ResponseField>
</Expandable>
</ResponseField>
